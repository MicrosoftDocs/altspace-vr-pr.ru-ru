# <a name="using-the-multimedia-console"></a>Использование консоли мультимедиа

Консоль мультимедиа — это средство, которое обеспечивает общий доступ к мультимедиа в событиях и мировых средах. Его можно использовать для обмена такими объектами, как изображения, слайды презентации, ливестреамс, видео, списки воспроизведения и многое другое. Ниже приведена пошаговая инструкция по использованию мультимедийной консоли **v 0.5.0 +**. 

## <a name="getting-started"></a>Начало работы

Начало работы с консолью мультимедиа состоит из двух этапов.  Сначала веб-портал, который будет использоваться для создания и публикации конфигурации для сеанса консоли мультимедиа, размещенного в вашей среде.  Вторым является размещение приложения консоли мультимедиа в среде и Задание кода конфигурации, который должен использоваться.

### <a name="configuring-the-multimedia-console-with-the-web-portal"></a>Настройка консоли мультимедиа с помощью веб-портала

1. Во-первых, необходимо убедиться в том, что содержимое размещено в сети, так как вам понадобится URL. (Вы можете отправить фотографии в altvr.com, разместить видео .mp4 файл в сети или использовать ссылку Дливе Live Stream: https://dlive.tv/yourlivestream) 
2. Перейдите на веб-портал для консоли мультимедиа по адресу [https://multimedia-console.altvr.com/](https://multimedia-console.altvr.com/)
3. На веб-портале можно создать и опубликовать конфигурацию для консоли мультимедиа.  (Дополнительные сведения о различных свойствах см. ниже.)
4. После того как вы ввели носитель в список носителей и настроили общие параметры, нажмите кнопку Опубликовать в правой верхней части приложения.
5. После завершения публикации появится диалоговое окно с двумя текстовыми файлами для входа в консоль мультимедиа, которую вы поместили.
  
### <a name="placing-the-multimedia-console-in-your-environment"></a>Размещение консоли мультимедиа в своей среде

1. Выберите на **> редакторе мира панель редактора > приложения пакета SDK > консоль мультимедиа**. (Не переходите в **мир > основы > приложение пакета SDK**— это для незарегистрированных приложений.)  
2. Разместите консоль мультимедиа для лучшего набора своих сфер и аудитории.
3. Чтобы выйти из режима редактирования, нажмите кнопку "оранжевый режим изменения".
4. Вам будет предложено сделать **вас владельцем проигрывателя мультимедиа?** Если вы являетесь пользователем, который является официальным владельцем этого сеанса консоли мультимедиа, подтвердите его и продолжайте. (Также доступны и другие разрешения ролей. Подробный список см. ниже.)
5. Выберите Да, чтобы подтвердить, что вы являетесь основным узлом.  
6. Откроется диалоговое окно с запросом на ввод кода с веб-портала или допустимого JSON.  Введите два слова на веб-портале, включая тире, и нажмите кнопку ОК. (JSON — это расширенная конфигурация, описанная ниже).
7. Консоль мультимедиа должна быть загружена через несколько секунд с конфигурацией, созданной на веб-портале.

### <a name="controlling-the-multimedia-console"></a>Управление консолью мультимедиа

1. После ввода кода и завершения процесса настройки вы увидите, что кнопки управления отображаются под медиа-дисплеем. 
    * **Воспроизвести** запускает средство просмотра мультимедиа (или перезапускается в текущей записи, если ранее она была остановлена). 
    * **Остановить** останавливает средство просмотра мультимедиа и скрывает Текущий носитель.  
    * **Следующий/** предыдущий переход к следующему или предыдущему носителю 
    * **x/x**   показывает текущий индекс в списке носителей и позволяет перейти к любой точке в списке
    * **Config** позволяет повторно ввести новый код с веб-портала, чтобы задать новую конфигурацию в консоли. 

Теперь вы настроили начало совместного использования через консоль мультимедиа!  
 
## <a name="working-with-the-web-portal"></a>Работа с веб-порталом

Веб-портал — это веб-приложение, которое позволяет настраивать различные функции консоли мультимедиа.  Эти функции подразделяются на две категории. Общие параметры консоли мультимедиа и список воспроизведения мультимедиа.

### <a name="multimedia-console-general-settings"></a>Общие параметры консоли мультимедиа

**Playback Settings**

Общие параметры воспроизведения для списка носителей

* **Список носителей циклов**— определяет, должен ли список носителей прокручиваться сразу после достижения конца списка.
* **Метод Start** — выбирает метод запуска консоли мультимедиа.
    * Ручная — Ожидание нажатия кнопки воспроизведения перед запуском мультимедиа
    * Автоматический запуск с начала — автоматический запуск списка носителей с начала списка
    * Автоматический запуск случайным образом запускает носитель с произвольной начальной точки в списке

**Роли**

Назначения ролей для управления и настройки консоли мультимедиа.    Эти роли разбиваются на следующие наборы:

* **Только владелец** — пользователь, являющийся владельцем сеанса консоли мультимедиа.
* **Пользователи с повышенными правами** — пользователи, имеющие роли модератора или ведущего приложения в пространстве, на котором настроена консоль мультимедиа изначально
* **Все пользователи** — все пользователи

Эти роли помещаются в том смысле, что все роли выше, выбранные в этом списке, также получат разрешение на использование этой функции.  Пример: **Пользователи с повышенными привилегиями** включают **владельца** , даже если они не являются модератором или узлом * * в алтспацевр. Ниже перечислены функции, контролируемые назначениями ролей.

* **Может управлять проигрывателем мультимедиа** — определяет, какие роли могут управлять кнопками воспроизведения мультимедиа для консоли мультимедиа.
* **Можно настроить проигрыватель мультимедиа** . определяет, какие роли могут настраивать консоль мультимедиа, путем предоставления доступа к кнопке **конфигурации** .

### <a name="adding-photos-and-videos-to-the-media-list"></a>Добавление фотографий и видео в список мультимедиа

Носитель — это сердце консоли мультимедиа.  Изображения и ссылки на видео поддерживаются как типы мультимедиа в консоли мультимедиа.  Чтобы добавить новый носитель, выберите значок **Добавить изображение** или **добавить значки видео** , чтобы открыть диалоговое окно для ввода сведений о файлах мультимедиа и параметров.  Ниже приведена классификация типов мультимедиа и связанных с ними параметров.

**Изображение**

Изображения должны быть стандартными типами изображений, такими как JPEG, PNG и наверх. Они должны размещаться в любом месте с общедоступной ссылкой.

* **Имя — (** обязательно) имя, с которым нужно указать образ.
* **URL-адрес изображения** — (обязательно) общедоступный URL-адрес изображения
* **Пропускать после** — количество секунд, в течение которых изображение должно быть пропущено после

**Видео**

Видео могут быть размещенными видео или интерактивными потоками через Твитч и Дливе.  (Другая поддержка может работать с дополнительной работой для получения соответствующего URL-адреса потока, но не полностью поддерживается в консоли мультимедиа).

* **Имя — (** обязательно) имя, с которым вы хотите опознать видео.
* **URL-адрес видео** — (обязательный) общедоступный URL-адрес, на котором размещается видео, или поток, из которого он обрабатывается.
* **Пропускать после** — количество секунд, в течение которых видео должно быть пропущено после

> [!NOTE]
> ОБЯЗАТЕЛЬНО. Укажите время, соответствующее длине видео, чтобы обеспечить правильную пересылку видео. Например, если видео находится в течение 5 минут после 300 секунд, в противном случае видео не будет переходить к следующему фрагменту содержимого.

* **Том** — объем видео в диапазоне от 0 (мин) до 1 (максимум).
* **Время начала** — количество секунд с начала видео.
* **Начальное расстояние начала** — расстояние в метрах в мире, на которое начинается отключение тома при переходе с консоли мультимедиа
* **Действие при завершении видео** — действие, которое будет выполнено после окончания этого видео.
    * Остановить — список носителей останавливается после завершения видео
    * Цикл — видео пройдет цикл до тех пор, пока не будет пропущено вручную
    * Воспроизвести далее. следующий носитель в списке носителей будет запущен после окончания текущего видео.

## <a name="working-with-json-directly-advancedoptional"></a>Непосредственная работа с JSON (дополнительно или необязательно)

Консоль мультимедиа поддерживает ввод JSON непосредственно в запрос консоли в Алтспацевр.  JSON — это внутренний механизм, с помощью которого мы включили конфигурации проигрывателя мультимедиа. Предоставление возможности непосредственного задания JSON — это то, что позволяет более опытным пользователям создавать собственные рабочие процессы, которые задают свои потребности и знакомство с JSON.  Ниже приведено краткое описание структуры JSON и схемы, с помощью которой проверяется JSON-код. Более подробное описание приведенных ниже свойств см. в разделах, посвященных настройке консоли мультимедиа.  В этом разделе основное внимания уделяется примерам схемы и структурированию данных JSON.

### <a name="global-media-settings"></a>Глобальные параметры носителя

```json
{
  "loopMediaList": true | false
  "startMethod": "manual" | "autostart-beginning" | "autostart-random"
  "controlMediaPlayer": "everyone" | "elevated" | "owner"
  "configureMediaPlayer": "elevated" | "owner"
  ...
}
```

### <a name="media-list"></a>Список носителей

Список носителей — это свойство, заданное в корне структуры JSON, например роли и параметры воспроизведения.  Это простой массив, который может содержать одну из следующих структур конфигурации носителя. (Дополнительные сведения о том, что они представляют, см. в описании свойств выше).

**Пример изображения**

*Обязательные поля: "Name" и "imageUrl"*

```json
{
    "name": "Altspace Screenshot",
    "imageUrl": "https://pbs.twimg.com/media/CxJ-fJqUsAAFtd9.jpg",
    "skipAfter": 10
}
```

**Пример видео**

*Обязательные поля: "Name" и "Видеаурл"*

```json
{
    "name": "Ninja Twitch Live Stream",
    "videoUrl":"https://www.twitch.tv/ninja",
    "volume":0.2,
    "startTime":0,
    "endOfVideoAction":"play-next"
}
```

### <a name="schema"></a>схема

```json
{
  "$schema": "https://json-schema.org/draft-04/schema#",
  "type": "object",
  "required": [
    "mediaList"
  ],
  "properties": {
    "loopMediaList": {
      "type": "boolean",
      "description": "Whether to loop through the media list when reaching the beginning or end of the list."
    },
    "controlMediaPlayer": {
      "type": "string",
      "enum": [
        "everyone",
        "elevated",
        "owner"
      ],
      "default": "owner",
      "description": "What roles are able to control the media player. (Owner can always control player)"
    },
    "configureMediaPlayer": {
      "type": "string",
      "enum": [
        "elevated",
        "owner"
      ],
      "default": "owner",
      "description": "What roles are allowed to configure the media play list.  Note: This role needs to be able to control the media player in order to configure it. (Owner can always configure media)"
    },
    "startMethod": {
      "type": "string",
      "enum": [
        "manual",
        "autostart-beginning",
        "autostart-random"
      ],
      "default": "manual",
      "description": "The method by which the media player should start"
    },
    "mediaList": {
      "description": "A list of images or videos to configure the media player to operate on.",
      "type": "array",
      "items": {
        "oneOf": [
          {
            "title": "Image",
            "type": "object",
            "description": "Configuration for an image media.",
            "properties": {
              "imageUrl": {
                "type": "string",
                "description": "The url for the image to load."
              },
              "skipAfter": {
                "type": "number",
                "minimum": 5,
                "default": null,
                "description": "The number of seconds that should pass before skipping to the next media. (Minimum 5)."
              }
            },
            "required": [
              "imageUrl"
            ]
          },
          {
            "title": "Video",
            "type": "object",
            "description": "Configuration for a video media.",
            "properties": {
              "videoUrl": {
                "type": "string",
                "description": "The url of the video to load."
              },
              "skipAfter": {
                "type": "number",
                "minimum": 5,
                "default": null,
                "description": "The number of seconds that should pass before skipping to the next media. (Minimum 5)."
              },
              "volume": {
                "type": "number",
                "minimum": 0,
                "maximum": 1,
                "default": null,
                "description": "The volume to play the video at. (Minimum 0, maximum 1)"
              },
              "startTime": {
                "type": "number",
                "minimum": 0,
                "default": null,
                "description": "The time in seconds from the start of the video to begin playing the video at. (Minimum of 0)"
              },
              "rolloffStartDistance": {
                "type": "number",
                "minimum": 0,
                "default": null,
                "description": "The distance in meters away from the media player that the volume will begin to fall off. (Minimum 0)"
              },
              "endOfVideoAction": {
                "type": "string",
                "enum": [
                  "stop",
                  "loop",
                  "play-next"
                ],
                "default": null,
                "description": "The type of action to take at the end of the video."
              }
            },
            "required": [
              "videoUrl"
            ]
          }
        ]
      }
    }
  }
}
```
